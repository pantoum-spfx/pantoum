AI Flow in PANTOUM

How Claude analyzes errors, migrates code, and verifies results

PANTOUM uses the Claude Agent SDK to give Claude direct access to your codebase. Claude can read files, apply edits, run builds, and search for patterns — all tracked with full transparency.

What Claude Sees and Does

Input

Claude receives a structured prompt containing:

  • The error output or migration instructions
  • The working directory (SPFx solution root)
  • Constraints: preserve functionality, batch edits, limit builds

Available Tools

📖Read
✏️Edit
✏️MultiEdit
🔍Grep
💻Bash
No Web Browsing
Claude operates exclusively on your local files. All migration instructions and reference documentation are embedded in the prompts — Claude cannot fetch external URLs, ensuring results are not influenced by compromised or outdated sources.

Execution Pattern

1. Read Examine error output and source files
▼
2. Plan Identify root causes and plan all fixes
▼
3. Fix Apply all edits in one batch (Edit / MultiEdit)
▼
4. Verify Run build to confirm fixes work

Every tool call is logged — files read, edits made, commands run

Build Error Fix Loop

When --aiFixBuildErrors true, Claude fixes compilation errors iteratively.

npm run build Run the build command
▼
Build Fails TypeScript errors detected
▼
Max 3 iterations (configurable via --aiMaxRetries, range 1-10)
Claude Analyzes Errors Reads error output + source files
▼
Generates & Applies Fixes Edit / MultiEdit on affected files
▼
Re-run Build Verify fixes work
▼
Pass
or
Fail → retry

Key Constraints

  • Read all error files first, then fix in batch
  • Run build at most 3 times total
  • Error count must decrease to continue retrying
  • Preserve existing business logic
  • Structured prompts for consistency

Common Fix Types

  • TS2564 — Add ! definite assignment
  • TS2532 — Add ?. optional chaining
  • TS2345 — Add type assertions
  • ESLint — Bulk disable rules if 10+ warnings

Package Migration

For major version upgrades (PnP v3→v4, MGT deprecation), Claude performs complete code transformations.

1. Detection
Manual steps in pantoum.patches.yml trigger migration when requiresMigrationAnalysis: true. Each migration has an AI context that loads a template file from src/templates/*.md with all instructions.
Triggered by: PatchService
2. Transformation
Claude receives a migration-specific prompt with patterns to update. Reads source files, applies import rewrites, API changes, and config updates using Edit/MultiEdit.
Tools: Read, Edit, MultiEdit, Grep, Bash
3. Self-Verification
Claude runs grep commands to check that old patterns are fully eliminated. If patterns remain, applies targeted fixes and re-verifies. Up to 3 iterations.
Tools: Grep, Bash, Read (verify) → Edit (fix)

Supported Migrations

PnP JS v3 → v4 Import changes, .data removal, auth updates
MGT Deprecation Package replacement, lazy loading
React Lifecycle UNSAFE_ methods → modern equivalents
Fluent UI v7 → v8 office-ui-fabric-react → @fluentui/react
Gulp → Heft Scripts Build system migration (SPFx 1.22+)
Gulpfile Custom Logic Gulp to Heft with env injection
SCSS Declaration Order Declarations before nested rules

Self-Verification Loop

After migration, Claude verifies that all old patterns were removed — with evidence.

Verify → Fix → Repeat (max 3×)
Verify Run grep for each old pattern
Report: found or not found
▼
All passed → done
or
Issues found
▼
Fix Apply targeted edits at exact file:line locations
↻ loop

Verification Checks Example (PnP v4)

# Pattern that should NOT exist after migration: grep -rn '.data.ID' src/ --include="*.ts" Expected: No matches grep -rn '@pnp/common' src/ --include="*.ts" Expected: No matches grep -rn 'presets/all' src/ --include="*.ts" Expected: No matches

Claude must show actual grep output as evidence — not just claim patterns are gone.

Verification Summary
✔ VERIFIED: .data.ID removed
✔ VERIFIED: @pnp/common replaced
✘ NOT_VERIFIED: presets/all in 2 files
→ Fix iteration triggered

Claude Agent SDK Integration

PANTOUM uses the official @anthropic-ai/claude-agent-sdk for all AI operations.

Fluent API

claude() .withModel('claude-sonnet-4-6') .inDirectory(solutionPath) .allowTools('Read', 'Edit', 'Grep', 'Bash') .withTimeout(600000) .withSessionId('pantoum_migration_...') .withDebugFile('debug.jsonl') .onToolUse((tool) => { /* track */ }) .onAssistant((text) => { /* log */ }) .skipPermissions() .query(prompt)

Authentication

Works with Claude Code subscription (no API key needed) or ANTHROPIC_API_KEY for direct API access.

Models

Sonnet 4.6 (default)
Fast, capable for most migrations
Opus 4.6 (optional)
Best for complex multi-file migrations
Haiku 4.5 (lightweight)
Lightweight scoring tasks

Key Design Choices

  • Structured prompts for consistent error fixing
  • Evidence-based verification (grep output, not claims)
  • Custom session IDs for report correlation
  • SDK debug trace (JSONL) for execution analysis
  • Full audit trail in reports and debug files
